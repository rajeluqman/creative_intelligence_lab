## Proven vs claimed (the Volve lesson)
During the pipeline_retrofit ground-truth audit, Volve's README claimed "84 tests, 0 failures"
and "Phase 7 Done" while the actual test/DQ directories were empty stubs — a real repo,
consistently described optimistically, not maliciously, but wrongly. This section exists to
force the same check every time:

| Claim (in README/resume/docs) | Evidence (`file:line` or command output) | Status |
|---|---|---|
| | | proven / unverified / false — correct the claim |

Every bullet in a README's "Results" section or a resume claim tied to this project should have
a row here. If it can't get a `file:line`, it's not proven — soften the claim instead of leaving
it.
